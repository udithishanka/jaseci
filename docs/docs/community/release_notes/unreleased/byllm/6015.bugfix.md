- **Fix: streaming generator crashed with "Token was created in a different Context" when consumed from a thread-pool worker**: `Model.invoke()` returned a streaming generator whose `finally` clause reset the `byllm_invocation_id` ContextVar via a token bound to the caller's `contextvars.Context`. Drivers that iterate the generator from a different Context, such as `jac-scale`'s SSE microservice running `loop.run_in_executor(None, _next_or_stop, gen)`, raised `ValueError: Token was created in a different Context` on cleanup and killed the stream (the UI received no chunks). The wrapper now captures its own `Context` with the invocation id pre-set and drives each `next()` via `ctx.run(next, gen)`, so iteration is safe from any executor and nested `by llm()` calls during streaming still observe the correct parent invocation id.
