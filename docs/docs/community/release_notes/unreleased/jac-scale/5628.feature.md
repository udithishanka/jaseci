- **Feature: Distributed coordination primitives (`jac_scale.coord`)**: New module exposing Redis-backed primitives for multi-pod apps — `acquire_lock` / `release_lock` / `distlock` (with fencing token + Lua atomic release), `kv_set` / `kv_get` / `kv_delete` / `kv_exists` / `kv_incr` (TTL-keyed store), `publish` / `subscribe` / `unsubscribe` / `on` (pub/sub with process-wide fan-out). `@restspec` now honors `lock`, `lock_timeout_ms`, `publish_on_entry`, `publish_on_success` kwargs — the walker callback wraps the dispatch with lock acquire/release and publishes events at the documented points. Keys are auto-prefixed `jac_coord:<JAC_APP_NAME>:` and gracefully degrade (or fail) based on `JAC_COORD_ON_FAILURE=degrade|fail`. Closes #5628 (minus streams, rate-limit, and cache; tracked for follow-up).
