- **Feature: Code-injection microservice deploy**: `[plugins.scale.microservices].code_inject` deploys the `sv import` fleet from a base image + code-server (the monolith Dockerless path, fanned out per sibling via `JAC_SV_NAME`) with no per-app image build, so code-sync platforms (jacBuilder/jachammer) can deploy a microservice app.
- **Feature: Namespace-correct peer DNS**: every fleet pod gets `POD_NAMESPACE` via the downward API, so `in_cluster_namespace()` resolves the real namespace and peer DNS (`<svc>-service.<ns>.svc.cluster.local`) works outside the `default` namespace.
- **Feature: Injectable microservices config**: `MicroserviceManifestBuilder` and `KubernetesMicroserviceTarget` accept an injected `microservices_config`, so a platform can deploy a different app's fleet instead of reading its own process's jac.toml.
- **Feature: Code-injection pod defaults**: code-injection pods install jaseci and build the client at runtime, so they get a startup probe (liveness/readiness suppressed until the app first responds, instead of being SIGKILLed mid-boot) and a 2Gi default memory limit (the in-pod client build OOMs at the 1Gi service default).
- **Fix: Streaming gateway passthrough**: the gateway's built-in `/function/*` and `/walker/*` passthrough now streams responses via `stream_forward` (status-only fan-out) instead of buffering the whole body via `raw_forward`, so SSE from a client-served generator (e.g. `guestbook_story`) arrives frame-by-frame instead of all at once.
