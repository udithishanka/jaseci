Microservice K8s deploy is now usable from code-injection platforms (jacBuilder/jachammer): (1) `[plugins.scale.microservices].code_inject` deploys the `sv import` fleet from a base image + code-server (the monolith Dockerless path, fanned out per sibling via `JAC_SV_NAME`) with no per-app image build; (2) every fleet pod gets `POD_NAMESPACE` via the downward API so peer DNS resolves in any namespace, not just `default`; (3) `MicroserviceManifestBuilder` and `KubernetesMicroserviceTarget` accept an injected `microservices_config`, so a platform can deploy a different app's fleet instead of reading its own process's jac.toml; (4) code-injection pods install jaseci and build the client at runtime, so they get a startup probe (liveness/readiness are suppressed until the app first responds, instead of being SIGKILLed mid-boot) and a 2Gi default memory limit (the in-pod client build OOMs at the 1Gi service default).
