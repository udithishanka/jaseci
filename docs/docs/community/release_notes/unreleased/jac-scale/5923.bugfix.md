- **Fix: microservice gateway can no longer brick itself on slow startup**: `wait_for_health` no longer hardcodes a 20s startup timeout. The new `[plugins.scale.microservices.services.NAME].startup_timeout` (and global `[plugins.scale.microservices].startup_timeout`) lets apps with slower boot (warmpool prefill, byllm import, first-run jac compile) raise the bar, defaulting to 60s. `ServiceProcessManager` also spins up a background recovery monitor that periodically re-runs `check_all_health`, so a service that crosses the startup bar by a few seconds is flipped back to `HEALTHY` automatically instead of staying `UNHEALTHY` for the lifetime of the gateway. The orchestrator starts the monitor right after the initial wait loop and stops it cleanly on shutdown. Symptom this fixes: gateway returning `Not found on any service` on `/walker/*` and `503 Service Unavailable` on `/api/<svc>/*` even though the underlying service is answering `/healthz` on its port.
