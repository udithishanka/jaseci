- **Fix: `--no-image` bootstrap installs the scale plugin's runtime deps + `jac-client`**: the in-pod bootstrap installed bare `jac-scale`, so the `jac-scale:scale` plugin failed to load in every pod (gateway crashed on `jac scale gateway`, service pods 404'd on `/healthz/*` and never went Ready). It now installs `jac-scale[all]` plus `requests` (a scale-plugin runtime dep that jac-scale ships only in its `test` extra, so `[all]` alone misses it), and `jac-client` when present locally so fullstack `sv import` apps can serve their client.
