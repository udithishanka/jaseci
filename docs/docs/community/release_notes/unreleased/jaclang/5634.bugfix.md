- **Fix: base class names resolve in enclosing scope**: Base class expressions in `obj Child(Parent)` are now looked up against the Archetype's enclosing scope instead of its own symbol table, matching Python semantics. Previously, an inner member that shadowed the base identifier (e.g. a `def Parent` inside `Child`) was found first, collapsing the MRO to `[Child]` and hiding inherited members. Real-world impact: `datetime.datetime.fromisoformat(...)` no longer reports a spurious E1030 because CPython's `class datetime(date)` has a `def date()` method that shadowed the outer `date` class. PEP 695 type parameters declared on the class remain visible inside the base list.
