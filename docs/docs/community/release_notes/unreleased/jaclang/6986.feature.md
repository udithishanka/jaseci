- **Feature: native programs can static-link a bundled C floor into the binary (#6978 Phase 0.1a)**: a clib floor import such as `import from lzma { def lzma_crc32(...) }` now links the library's static archive directly into the `nacompile` binary instead of carrying a runtime dylib dependency. New `ar` archive reader in `linker_common` (`parse_ar_archive`, `parse_ar_symbol_index`, `select_ar_members`) pulls only the transitively-needed members; `MachOLinker` gained multi-object merge (`archive_members`) with cross-object symbol resolution; and `nacompile` resolves the import to its bundled `lib<name>.a`, dropping it from the dylib dependency list. Verified on macOS arm64 (the TLS-free floor set): `import from lzma`/`import from expat` programs run with `otool -L` showing only libSystem. Mach-O only for now; ELF/PE keep the dylib path, and production payload staging of the archives is Phase 0.2.
- **Feature: `jac test` accepts multiple file paths**: `jac test a.jac b.jac` previously errored with "unrecognized arguments"; the path argument is now a multi-value positional (matching `jac check`/`jac lint`), while bare `jac test` and single-file runs are unchanged.
