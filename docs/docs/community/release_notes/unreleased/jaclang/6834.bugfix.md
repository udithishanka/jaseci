- **Fix: clib struct pointer subscript read/write now lowers correctly (#6707, #6708)**: `buf[i]` and `buf[i] = val` on a clib struct pointer parameter previously hit `E5090` ("subscript/index access not supported"). Native function annotations are unstamped by the type checker, so clib struct parameters like `buf: Color` lowered to `i64` in LLVM via the `_resolve_jac_type` fallback, and the subscript path's `ir.PointerType` check was never satisfied. `NaIRGenPass` now tracks `clib_local_struct_names: dict[str, str]` (populated from `p.type_tag.tag` at function entry and from typed local annotations like `c: Color = ...`), so the read path emits `inttoptr(buf_i64, %Color*) → GEP([i]) → ptrtoint → i64` and the write path emits `inttoptr + GEP` for the destination followed by the appropriate struct copy (`%Color*` boxed, `%Color` by-value, or `i64` pointer-as-int). Field access on the subscript result (`c.r`) works via the existing `_codegen_field_access` `IntType → inttoptr` path, now that `_struct_name_of` also consults `clib_local_struct_names` for unstamped `Name` nodes.
