- **Feature: `RestSpecs` + `ScheduleSpec` coordination fields**: `RestSpecs` gains optional `lock`, `lock_timeout_ms`, `rate_limit`, `rate_limit_key`, `publish_on_entry`, `publish_on_success`, `cache_ttl_ms`, `cache_key`, `stream`, `stream_max_len` — consumed by jac-scale's endpoint registration to wrap walker dispatch with the corresponding middleware. `ScheduleSpec` gains `leader` / `leader_key` / `leader_ttl_ms` for leader-elected background tasks. Defaults are `None`/`0`/`False` so existing `@restspec(method=..., path=...)` and `@schedule(interval=...)` usages are fully backward-compatible.
