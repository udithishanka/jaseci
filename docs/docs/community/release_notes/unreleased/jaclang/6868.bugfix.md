- **Fix: `jac nacompile` no longer mis-types unannotated locals bound to imported struct factories (#6868)**: When an entry module imported a struct factory whose return annotation the type checker had left unstamped (`_resolve_jac_type` saw `.type is None`), the native codegen defaulted the lowered type to `i64`. Locals bound to such imported factories were therefore lowered as scalars, and subsequent field access tripped `E5090` on a non-struct value. The lowering now falls back to the `struct_types` name registry before defaulting to `i64`, mirroring the existing lookup in `_lower_class_type`, so unannotated imports resolve to the correct struct pointer.
